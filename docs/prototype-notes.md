# 原型筆記(對照 GDD)

> 目的:用最小可玩版本找出這款遊戲的核心樂趣。
> 本文記錄實作範圍、共同定案的設計決策、所有調參旋鈕、已知簡化。

## 版本

- **v0.4(當前)「後勤魔王」** — 大改:玩家的手從「產量端」移到「加工端」。
  - 實玩發現 v0.1~0.3 的根本問題:**以量取勝、時間夠久遲早佔領**(GDD 警告過的「漂成放置型」)。
  - 三個結構修正同時上:①**兵源固定** — 異界不再是場上生產建築,軍團以固定速率從泉水歸返(傷兵、血量不一、種族=陣容組成),量不可放大;②**門會扎根** — 抵抗隨開門秒數成長(+0.02/s²,上限 3.0),每扇門是一個「征服窗口」,拖=輸;③**質=戰力** — 規格不符 ×0.05 純噪音,規格靠**加工站**達成(回復/附魔,蓋在線上、經過即生效、可串接)。
  - 征服異界的獎勵重綁定:**兵源加入該族 + 獲得該界的設施**(屬性界→附魔站、純種族界→回復站)。
- v0.3 — 泉水復活設施(取代原路折返);壓力調降(抵抗 1.2→0.8、單位值 1→1.5)。
- v0.2 — 手繪產線;傳送門隨機位置、分批撕開;全程即時+戰術暫停。
- v0.1 — 首版:固定門位、放置階段、點對點連線、異界生產建築。

## 怎麼跑

```bash
npm install
npm run dev        # http://localhost:8080
```

操作(全程即時):
- **放置**:點下方卡片(泉水⛲ / 加工站)→ 點棋盤格。點角落 ✕ 收回。加工站可直接蓋在既有的線上(改裝產線)。
- **畫線**:從**泉水**按住拖曳,沿格子畫到傳送門放開(自動繞障礙、可回拉擦除)。泉水最多 4 條線、輪流均分;同目的地重畫 = 取代。
- **拆線**:點線段。
- **節奏**:空白鍵 = 戰術暫停(仍可放置畫線),`1`/`2` 調速,ESC 取消。
- Console 有 `__game` 除錯把手。

**單位的一生(v0.4)**:從前線歸返 → 泉水復活(殘值 0.75~1.35,種族=陣容隨機)→ 沿線經過加工站(回復 +0.25/座、附魔換屬性欄)→ 灌入門(×規格符合度)→ 打 4 折 → 傳送回泉水(+0.15)再上工 → 折舊後 < 0.2 榨乾。反噬敵人沿線殺回,對沖、癱瘓加工站 4 秒。

## 為什麼量贏不了、耐心磨不了(v0.4 的三根柱子)

1. 兵源速率固定 → 你的總供給是**預算**,不是可放大的變數。
2. 門的需求隨時間**成長** → 預算/需求比只會惡化,等待嚴格虧損;早關門=便宜,晚關門=貴。
3. 你唯一能變出來的產能是**效率**:配對 ×1.0 vs 不符 ×0.05 — 加工網路的品質直接就是戰力。

## MVP 範圍對照

| 條目 | 狀態 |
|---|---|
| 單一場景 | ✅ 每關 `scene.restart()` 重生成 |
| 2–3 個起始異界 | ✅ 3 座 = 兵源組成 + 起始設施(火附魔/水附魔/回復×2) |
| 3 個傳送門、需求 0–2 軸 | ✅ 需求偏屬性(附魔可解),種族需求隨關數增加;需求屬性保證 ⊆ 玩家現有 |
| 前線推拉 + 反噬 | ✅ 單軸 −3/−1/0/+3 × 基數;抵抗+湧出隨門齡扎根成長 |
| 加工系統 | ✅ **前移為核心**(GDD 原定局外解鎖):回復站、火/水/時附魔站;經過即生效、可串接 |
| 穩定即過關/完整度/碎裂結算 | ✅ 外殼不變;征服=兵源+設施 |
| 獎勵點 → 永久解鎖 | ✅ localStorage;3 項解鎖 |
| 暫不做:貨幣、劇情、門內視覺化 | ✅ 皆未做 |

## 共同定案的設計決策(時序)

1. **快循環節奏**(基數 20,GDD 🔴 待拍)— v0.1。
2. **手繪路徑、裂隙隨機分批**(玩家要親手、湧現)— v0.2 實玩回饋。
3. **泉水統一復活**(取代同線折返/對門出口)— v0.3 討論定案。
4. **玩家操作移到加工端**(產量端沒有玩法)— v0.4 實玩回饋+討論定案。
5. **附魔規則**:單位 = [種族(先天), 屬性欄(1 格、新蓋舊)]。
6. **征服獎勵**:兵源加入該族 + 該界設施一座;基礎另送回復站 ×1。
7. 對沖用原始戰力(不吃規格權重)— 待驗證是否要改。
8. 敵人只沿「有連線」的門湧出;走到泉水散逸(還不會破壞泉水本體)。

## 調參旋鈕(全部在 `src/game/balance.ts`)

| 旋鈕 | 現值 | 說明 |
|---|---|---|
| `BASE` | 20 | 基數 = 主旋鈕(GDD 暫定 100 的快循環版) |
| `SPAWN_PERIOD` | 0.8s | 兵源歸返速率(固定,玩家不可放大) |
| `ARRIVAL_MIN/MAX` | 0.5~0.9 | 歸返傷兵殘值(× UNIT_VALUE) |
| `UNIT_VALUE` | 1.5 | 單位滿值 |
| `DOOR_DRAIN` | 0.8/s | 門基礎抵抗(+0.1/關) |
| `DRAIN_GROWTH` | 0.02/s² | **扎根成長**:開門每秒抵抗 +0.02(1 分鐘 +1.2) |
| `DRAIN_ABS_MAX` | 3.0/s | 單門抵抗上限 |
| `MATCH_MIN_W` | 0.05 | 不符 ≈ 純噪音;半符 0.525 |
| `HEAL_STATION` | +0.25/次 | 回復站(不回滿,串多座=時間成本) |
| `STATION_DISABLE` | 4s | 敵人癱瘓加工站的時長 |
| `PASS_MULT` / `CULL_VALUE` | 0.4 / 0.2 | 過門折舊/榨乾線(須 > 泉水回血不動點 0.25) |
| `SPRING_HEAL` / `RIFT_TRANSIT` | 0.15 / 0.5s | 復活小補/傳送延遲 |
| `SPRING_MAX_LINES` | 4 | 泉水線數(唯一源頭) |
| `ENEMY_PERIOD` | 2s | 反噬湧出基準;隨扎根程度加快(下限 0.8s) |
| `RIFT_FIRST/GAP/JITTER/LEAD` | 5/22/±6/4s | 裂隙節奏(隨關數壓縮) |

手感速算(v0.4):傷兵流平均 ~1.05 值 × 1.25/s ≈ 1.3/s 原料;經回復站可拉到 ~1.8/s;門從 0.8/s 起每分鐘 +1.2 → **一條好線大約有 60~90 秒的征服窗口**,兩線齊推 ~25 秒關門。無人餵的門約 23s 反噬、~55s 崩潰(扎根加速下沉)。

## 已知簡化 / 技術債

- 泉水是唯一源頭 → 多條線在泉水附近重疊;拆線點擊拆「先建的那條」。
- 快速斜向拖曳補格會出現階梯線;慢畫=你畫的形狀。
- 加工站觸發以「最近格心」判定,單位過站瞬間生效(無動畫)。
- 附魔對「回程撤退」單位不生效(僅出征方向)。
- 速率箭頭取樣 0.5s,偶爾閃爍。音效 0;無教學。
- 泉水收回 = 全場我方單位散逸(失去歸屬),屬激進簡化。
