# 原型筆記(對照 GDD v0.1)

> 目的:用最小可玩版本回答 GDD 的核心問題 ——
> **「鋪產線餵門 + 反噬時即時重新分配」這個 30 秒循環本身好不好玩?**
> 本文記錄實作範圍、我(Claude)代為裁決的設計點、所有調參旋鈕、已知簡化。

## 版本

- **v0.3(當前)** — ①新增**泉水**(玩家放置的復活設施,一關一座):單位過門後統一從泉水復活(×0.4 折舊 +0.15 小補),沿泉水的線再上工;**原路折返徹底移除**,沒泉水就餵完消散。這是第一座設施,為未來回復/加工設施鋪路。②依實玩回饋放緩壓力:門抵抗 1.2→0.8、單位值 1→1.5(反噬太快、餵門值太低);榨乾線 0.1→0.2(必須高於泉水回血不動點 0.25,否則單位永生)。
- v0.2 — ①產線改為**手繪拖曳**;②傳送門**隨機位置、分批撕開**,拿掉「放置階段→啟動」,改為**全程即時**+戰術暫停。
- v0.1 — 首版:固定門位、放置階段、點對點連線。

## 怎麼跑

```bash
npm install
npm run dev        # http://localhost:8080
```

操作(全程即時,可隨時做):
- **放置**:點下方卡片(異界或**泉水⛲**)→ 點棋盤格。點角落 ✕ 收回。泉水一關一座、不佔部署上限。
- **畫線**:從**異界或泉水**按住拖曳,沿格子畫到傳送門放開(會自動繞過障礙、可回拉擦除)。每個源頭最多 2 條,輪流均分;對同一扇門重畫 = 取代舊線。
- **拆線**:點產線任一線段。
- **節奏**:空白鍵 = 戰術暫停(暫停中仍可放置、畫線),`1`/`2` 調速,ESC 取消。
- 瀏覽器 console 有 `__game` 除錯把手。

**單位的一生**:異界出生(1.5)→ 沿線餵門(×匹配權重)→ 打 4 折 → 裂隙傳送 0.5s → **泉水復活**(+0.15,保留特性)→ 沿泉水線再上工 → 折舊後 < 0.2 榨乾消散(約 3 趟)。沒放泉水或泉水沒連線 = 餵完消散/在泉水待命。

## 一關的流程

開局棋盤空無一門 → 裂隙先「預告」`RIFT_LEAD` 秒(脈動圈,該格不可放置)→ 撕開成門,開始被抵抗下壓 → 之後每隔 `RIFT_GAP`±抖動 秒再撕一道,共 3 道(位置隨機,會避開既有門、異界與產線)。全部門解決(鎖定/崩潰)即過關。若場上已無活門但還有裂隙沒開,會提前撕開避免乾等。

## MVP 範圍對照

| GDD MVP 條目 | 狀態 |
|---|---|
| 單一場景 | ✅ `Game` 場景(每關 `scene.restart()` 重生成) |
| 2–3 個起始異界 | ✅ 3 座(龍焰🐉🔥/亡骸💀/泉妖🧚💧),解鎖後 +荒牙🐺 |
| 3 個傳送門、單軸或雙軸需求 | ✅ 需求數隨關數:L1 `[0,1,1]` → L4+ `[2,2,2]`;位置隨機、分批出現 |
| 前線推拉 + 反噬 | ✅ 單軸 −3/−1/0/+3 × 基數;反噬時敵人沿「有連線的產線」湧出 |
| 穩定即過關 | ✅ 全門非 active 且裂隙出完,即結算;征服異界加入陣容 |
| 完整度 1 次失敗即碎 | ✅ 預設 1,局外解鎖可到 3(GDD 上限) |
| 最小獎勵點 → 永久解鎖循環 | ✅ 1 征服 = 1 點,碎裂時入庫(localStorage);3 項解鎖 |
| 暫不做:加工/貨幣/劇情/門內視覺化 | ✅ 皆未做 |

## 我代為裁決的設計點(請過目,不同意就改)

1. **🔴 節奏味道(GDD 待決第一項)→ 選了「快循環」**:基數 20(非 100)、產速 1 隻/0.8s(非 1/2s)、門抵抗 1.2/s。專注餵一門約 30–60 秒穩定;無人餵的門約 17 秒反噬、50 秒崩潰。要回慢經營只動 `balance.ts` 三個數字。
2. **繞回重用 → 泉水復活(v0.3 與使用者共同定案)**:GDD 原文「從對門出來」曾誤做成同線折返;討論後定為**泉水統一復活** — 玩家放置、位置自選,等於把補給迴路的錨點交給玩家(門的位置是隨機的,錨點不能也交給 RNG)。泉水的流是**混特性**的(各族復活後混編),泉水線接哪扇門是新的決策點,也為未來加工站(重新貼標)留了戲份。線被拆時在途單位仍會半透明撤退回源頭待命。
3. **路上對沖用原始戰力**(不吃規格權重)— GDD「符合度=貢獻/推力權重」兩讀取寬鬆版:龍在哪條線上都是滿戰力,但只有對的門收貨才值錢。要改硬派版:`simStep` 對沖處乘 `matchWeight`。
4. **每關部署上限**(min(6, 2+關數))代理「場地空間有限」,防止異界收集雪球化;真正的空間爭奪(路徑佔格、加工佔格)留給加工版本。目前產線可互相交叉、也可穿過異界所在格以外的任何空格。
5. **敵人只沿「有連線」的產線湧出**;無連線的門靜默流失。敵人走到產線源頭即散逸,不攻擊異界本體(GDD 未定義,取最簡)。
6. **第 1 關保證所有門可全符** — 實測發現自由生成的門可能完全不符陣容(0.2 倍餵不動),配上完整度 1 會讓新手被 RNG 判死;自由門(收集新特性的誘因)自第 2 關開始。
7. **征服獎勵 = 該門背後的異界**,特性涵蓋門的需求(reqs ⊆ worldTraits)。
8. **抵抗隨關卡 +0.1/s(上限 2.0)、湧出間隔隨關縮短、裂隙間隔隨關壓縮**(GDD 待決「反噬是否隨關卡升高」→ 溫和版)。
9. **手動「粉碎裝置」**(HUD 右上,二次確認):自願結束此輪拿點數。GDD 沒寫;測局外循環很方便,先留著。

|---|---|---|---|
| `BASE` | 20 | 100 | 基數 = 主旋鈕。1 格 = 基數 × 比例 |
| `MAKER_PERIOD` | 0.8s | 2s | 異界產速 |
| `DOOR_DRAIN` | 0.8/s | 1/s | 門抵抗(+`DRAIN_PER_LEVEL`×關數);v0.3 由 1.2 調降 |
| `UNIT_VALUE` | 1.5 | 1 | 單位基準貢獻;v0.3 調升 |
| `PASS_MULT` | 0.4 | 0.4 | 過門折舊 |
| `CULL_VALUE` | 0.2 | — | 榨乾線;必須 > 泉水回血不動點(0.4x+HEAL 收斂於 0.25) |
| `SPRING_HEAL` / `RIFT_TRANSIT` | 0.15 / 0.5s | — | 泉水復活小補/過門→復活的傳送延遲 |
| `MATCH_MIN_W` | 0.2 | —(「低很多」) | 不符規格的貢獻權重;半符 0.6 |

手感速算(v0.3):1 條全符產線淨 +1.0/s 出頭 → 單線約 60 秒穩一門;接上泉水迴路(0.75/0.45 兩代復活)再快約三成;無人餵的門 25 秒反噬、75 秒崩潰;不符(0.2×)仍餵不動,必須換源或棄門。單位約跑 3 趟榨乾(GDD:2~3 趟)。

## 已知簡化 / 技術債

- 快速斜向拖曳時,補格用 BFS → 會出現階梯狀線;慢慢畫就是你畫的形狀。
- 兩條線重疊的格子,點擊拆線拆到「先建立的那條」。
- 門進度的速率箭頭(▲▼)取樣 0.5s,到貨離散 → 箭頭偶爾閃爍。
- 撤退單位(線被拆時)不會被敵人攔截(同向同速追不上,幾何上自洽)。
- 門鎖定/崩潰時在途單位掉頭回源頭待命;屬可接受損耗。
- 泉水每關需重新放置(關卡重生成);敵人走到泉水只是散逸,還不會破壞設施。
- 泉水復活的傳送途中(0.5s)若泉水被收回,該批單位散逸。
- 音效 0;無教學,僅文字提示。存檔只有獎勵點與解鎖,輪中不可存檔續玩。

## 素材需求(目前不需要)

現在全是程式繪製(幾何 + emoji),驗證玩法足夠。若進到「味道」階段,優先序:
1. 傳送門「裂隙擴大→撕開→反噬→崩潰」的動態表現(情緒核心)
2. 各種族小單位(頂視角小圖示,一種族一張)
3. 反噬警報、過門、對沖的音效三支
