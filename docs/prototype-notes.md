# 原型筆記(對照 GDD)

> 目的:用最小可玩版本找出這款遊戲的核心樂趣。
> 本文記錄實作範圍、共同定案的設計決策、所有調參旋鈕、已知簡化。

## 版本

- **v0.5(當前)「詛咒行軍」** — 概念重立:**倒置塔防**,當新概念設計、不揹舊版包袱。
  - 起因:多版概念混雜 —— v0.2 空間系統只剩殘骸、v0.4 經濟沒有操作面、反噬零反制。以塔防五拍逐格對照重建(見 GDD 核心循環),缺的兩格補上:**可花的獎勵**(魔力)與**端點懲罰**(永久死亡)。
  - 一關 = 一道門(Boss HP);固定小隊循環出擊;**詛咒 = 死亡來源**(線上持續扣血);**HP 是唯一貨幣**;魔力(過門貢獻換得)蓋站/升級/補員;過關 +1 技能點(局內技能樹);全滅 = 遠征潰滅 → 局外點。
- v0.4.1 — 固定佈局(泉水下中、單門上中、自動直線產線);移除裂隙隨機與手繪線。
- v0.4「後勤魔王」 — 玩家的手從產量端移到加工端(兵源固定、門會扎根、質=戰力)。
- v0.3 — 泉水復活設施。v0.2 — 手繪產線、裂隙隨機分批。v0.1 — 首版(異界生產建築)。

## 怎麼跑

```bash
npm install
npm run dev        # http://localhost:8080
```

操作(全程即時):
- **蓋站**:點下方卡片(回復 ✚80 / 火🔥 水💧 時⏳ 附魔 ✦100)→ 點棋盤格。**蓋在產線上才會生效**。
- **升級**:點場上的回復站(✦60 → ✦87;上限 Lv3)。附魔站不可升級。
- **拆除**:點站角落 ✕,退回累計投資的一半。
- **補員**:點「補員」卡(✦200),小隊未滿編時入隊一名滿血新兵。
- **節奏**:空白鍵暫停(仍可操作)、`1`/`2` 調速、ESC 取消放置。Console 有 `__game` 除錯把手。

**單位的一生(v0.5)**:滿血 100,開場散佈在泉水附近出發 → 線上每秒吃詛咒(L1 = 1.2/s)→ 經回復站 +30/50/70、附魔站換屬性 → 抵門:**貢獻 = 當前 HP × 乘子**(弱點 1.5/頑抗 0.6/中性 1),扣門血並換魔力(×0.5)→ 以「抵達 HP × 50%」於 0.8~2.4s 後(隨機)從泉水歸隊再出發 → 路上歸零 = **永久死亡**(小隊 −1)。右側「戰報」面板記錄每筆事件(轟門/死亡/歸隊/建設)。

## 為什麼這是「倒置塔防」

| 經典塔防 | 本作 |
|---|---|
| 敵人從入口進場(波次) | 小隊從泉水出發(固定隊,速率不可放大) |
| 塔對敵人做減法 | 設施對己方做加法(回復/附魔) |
| 擊殺 → 可花的獎勵 → 蓋塔/升級 | 過門貢獻 → 魔力 → 蓋站/升級/補員 |
| 漏怪到終點 → 扣命 | 死在路上 → 永久死亡(小隊就是命條) |
| 波次越來越兇 | 詛咒與門 HP 隨關數成長 |

## 調參旋鈕(全部在 `src/game/balance.ts`)

| 旋鈕 | 現值 | 說明 |
|---|---|---|
| `UNIT_HP` / `SQUAD_START` | 100 / 5 | 滿血/起始小隊(局外解鎖 +1) |
| `UNIT_SPEED` / `START_SCATTER` | 60px/s / 96px | 移速(單程 384px ≈ 6.4s)/開場散佈範圍 |
| `CURSE_DPS` +`CURSE_PER_LEVEL` | 1.2 +0.7/關 | 死亡來源;L1 單程約吃 7.7 血(與移速 100 時代相同) |
| `REVIVE_RATIO` | 0.5 | 復活 = 抵達 HP ×(技能樹 0.6/0.7) |
| `REVIVE_DELAY_MIN/MAX` | 0.8~2.4s | 歸隊延遲(範圍內隨機,打散出擊節奏) |
| `DOOR_HP` ×`DOOR_HP_GROWTH` | 500 ×1.35/關 | Boss 血量 |
| `TRAIT_LEVEL` / `WEAK_MULT` / `RESIST_MULT` | L3 / ×1.5 / ×0.6 | 門的弱點/頑抗(溫和乘子,非死門檻) |
| `MANA_RATE` | 0.5 | 魔力 = 門受傷 ×(技能樹 +25%) |
| `COST_HEAL/ENCHANT/RECRUIT` | 80/100/200 | 補員刻意貴:買質不買量 |
| `UPGRADE_BASE` ×1.45^(lvl−1) | 60 → 87 | 回復站升級費 |
| `HEAL_BASE` +`HEAL_PER_LVL` | 30 +20/級 | 回復站每次經過回血 |
| `SELL_REFUND` | 0.5 | 拆站退款(對累計投資) |

**手感速算(L1,無站)**:單程吃 7.7 詛咒 → 抵達 ~92 → 復活 46 → 38 → 19 → 11 → 死。一隻約貢獻 ~140,5 隻 ≈ 700 > 門 500 —— **裸打勉強能過、會折損;蓋回復站才有餘裕**(刻意的教學曲線)。魔力收入 ≈ 門血 × 0.5 ≈ 250/關 + 跨關結轉。移速 60 後一關 wall-time 約 2.5~3 分鐘,嫌慢用 2×。

## 共同定案的設計決策(時序)

1–8. v0.1–v0.4 舊定案(異界命名/推拉軸/泉水復活/加工端……)見 git 歷史版本。
9. **倒置塔防框架**(v0.5):塔防五拍逐格對照;玩家操作兵流側,設施=對己方做加法的塔。
10. **死亡來源 = 異界詛咒**:線上持續扣血、泉水是安全區;不做敵人實體/對沖(便宜、可讀,回復站直接對抗它)。
11. **固定小隊 + 魔力補員**:小隊就是預算與命;全滅且補不起 = 遠征結束(結束於「破產」而非瞬間)。
12. **全魔力經濟**:蓋站/升級/補員都花魔力;紅線 = 魔力**只能買質**,不可放大泉水速率(v0.4 祖訓)。
13. **跨關保留**:魔力、設施、隊員 HP 帶到下一關 —— 一次遠征是一條連續戰線,只換更硬的門。
14. **特性 = 溫和乘子**(補充條款):門的弱點 ×1.5/頑抗 ×0.6;不做 v0.4 的 ×0.05 死門檻;種族休眠、只玩屬性;L3 起出現。
15. **技能樹 = 局內、解鎖 = 局外**:過關 +1 點自由花(7 節點清單);潰滅結算獎勵點 = 通過關數。

## 已知簡化 / 技術債

- 單一直線產線,佈線玩法尚未回歸;站蓋在線外合法但無用(提示寫在托盤)。
- 附魔站不可升級;詛咒只隨關數成長、關內恆定(反拖延鈕先不開)。
- 隊員無個體名字/養成;補員永遠是滿血新兵;技能樹是清單不是樹狀圖(僅一條前置鏈)。
- 全滅但魔力足夠時,遊戲會停在「等你補員」——刻意不強制,軟鎖由玩家自己選。
- 音效 0;無教學;存檔 key 升到 `riftlord_save_v2`(v1 舊檔作廢)。
