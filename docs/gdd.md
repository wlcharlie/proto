## 一、願景 / 三大支柱

你是一覺醒來的**異界魔王**,靠**輿圖裝置**開啟通往各個異界的傳送門,把「生產線」鋪向異界並征服它們;一旦疏於照顧,傳送門就會反噬,**產線瞬間變戰線**。

支柱(迷路時回來看這三條):

1. **產線即戰線** — 同一套場上佈局,平時是征服用的生產線,失守時就是防禦用的戰場。這是本作的靈魂。
2. **異界即生產線** — 收集到的不是「兵」,而是你征服下來的**異界**(每個異界是一座生產來源)與加工選項。玩家把原始單位**組裝**成傳送門要的規格。
3. **碎裂即重來** — 裝置碎裂 = 一輪結束。帶著永久解鎖滾動變強(roguelike)。

## 二、核心循環

**基因定位(v0.5 定案):本作是「倒置塔防」** —— 玩家操作**兵流側**(如 Anomaly: Warzone Earth),設施是「對己方做加法的塔」。塔防五拍逐格對照:敵人進場 ↔ 小隊出發;塔削敵 ↔ 設施強化己方;擊殺賞金 ↔ 過門魔力;漏怪扣命 ↔ 路上永久死亡;波次變兇 ↔ 詛咒與門隨關數成長。

**關卡內(v0.5「詛咒行軍」):**
一關 = 一道**異界之門**(Boss HP 條)。固定**小隊**從**泉水**(安全區)依序出發,沿產線走向門;線上是**詛咒領域**(持續扣血,隨關數變強)
→ 沿途**設施**生效:回復站補血(可升級)、附魔站掛屬性(呼應門的**弱點/頑抗**,溫和乘子、不做死門檻)
→ 抵達門:**貢獻 = 當前 HP × 屬性乘子**,直接扣門血;同時獲得**魔力** = 貢獻 × 轉換率
→ 單位從泉水以「抵達 HP × 復活率」復活再出發 —— **HP 是唯一貨幣:詛咒扣它、回復站補它、門吃它、復活折它**
→ 線上 HP 歸零 = **永久死亡**(小隊 −1);魔力可**補員**(刻意貴)
→ 魔力三用途:蓋站/升級站/補員(**買質不買量**:泉水速率與出發節奏不可用魔力放大 —— v0.4 祖訓)
→ **門 HP 歸零 = 過關**:+1 技能點 → **局內技能樹**;魔力、設施、隊員 HP 跨關保留 → 下一道更硬的門
→ **全滅且魔力不足補員 = 遠征潰滅**(一輪結束)

**跨輪(局外):**
潰滅結算**獎勵點** = 通過關數
→ 用獎勵點做**局外解鎖**(起始小隊 +1、開局魔力、開局技能點)
→ 開新遠征

> **局內/局外定義:** 技能樹(過關拿點、潰滅清零)屬**局內**;獎勵點解鎖(跨遠征永久生效)屬**局外**。

## 三、核心機制

> **v0.5 注意:** 本節以下的「異界系統/傳送門規格/征服進度(推拉軸)/單位的一生」描述 v0.1–0.4 的模型,v0.5 已改為「門 = Boss HP、貢獻 = 傷害、弱點/頑抗 = 溫和乘子、詛咒 = 死亡來源」(見核心循環與 `docs/prototype-notes.md`)。保留當設計史;異界收集與種族層屬未來喚醒項。

### 異界系統(兵源與設施;v0.4 改版)

- **征服一個異界(穩定其傳送門)後,該異界即歸你所有**:①該族加入**歸返兵源的組成**(如「不死異界」→ 兵源混入不死族)②獲得該界的**設施**一座(屬性界 → 對應附魔站;純種族界 → 回復站)。**你擁有的異界 = 你的兵源組成 + 你的工具箱。**
- 異界不再是場上的生產建築(v0.1~0.3 曾是,實測產量端沒有玩法);前線輸送在畫面外進行,玩家專注泉水→門的後勤段。
- 一輪內累積擁有的異界,在裝置碎裂時清空(roguelike)。

### 加工系統(v0.4 前移為核心玩法)

- ~~透過局外解鎖取得,首版不做~~ → **v0.4 定案:加工就是本作的核心動詞**,取得方式改為**征服異界的設施獎勵**。單位**「經過即生效、不排隊」**,沿路徑經過即附加屬性/能力(火附魔、加翅膀會飛…),可串接多台。已實作:回復站(+0.25/次、不回滿)、火/水/時附魔站(屬性欄 1 格、新蓋舊;種族先天不可變)。

> 設計後果:沒有排隊/吞吐限制,所以玩法深度在於**「路徑佈線」(怎麼讓單位穿過需要的加工)**與**「產能分配」(把產出指向哪個門)**,而非平衡每秒流量。

### 傳送門

- 每門有**規格需求**:0–4 項角色特性(**0 項 = 任何單位皆可**,通常是早期/簡單的門;愈後期項數愈多)。
- **符合越多,效益越高**;不符合的單位仍有貢獻,但低很多。
- 每門只有**一條軸**:征服進度(= 前線位置),定義見下節。沒有另一條獨立的「穩定度」。

### 征服進度 = 前線(「餵」與「推」是同一件事)

**核心設計:每個傳送門只有一條軸——前線位置 = 征服進度。** 不另設「穩定度」,也不做獨立的戰鬥系統。

**四狀態軸(比例)+ 基數換算:**
門檻為「比例」,**1 格 = 基數 = 100 個單位貢獻**(基數 = 主旋鈕,調節點奏)。單位貢獻先定 **1**、敵方 **-1**(實測再調)。

| 比例   | 狀態     | = 累積淨貢獻 | 說明                                                     |
| ------ | -------- | ------------ | -------------------------------------------------------- |
| **+3** | 穩定     | **+300**     | 達標 → **鎖定**,不再反噬,可把產能轉去別的門。            |
| **0**  | 不穩定   | 0            | 新門開局值;往上推進中。                                  |
| **-1** | 開始反噬 | **-100**     | 敵方湧出、前線後退;**-1 到 -3 是「警告但可救」的緩衝**。 |
| **-3** | 崩潰     | **-300**     | 傳送門關閉(失敗 +1,即「保護魔王的斷尾」)。               |

> **手感速算(暫定數值):** 異界 **0.5 隻/秒**(1 隻/2 秒)、門抵抗 **-1/秒**。一個門淨速率 = 0.5 ×(餵它的異界數) − 1。→ **2 異界打平、3 異界才開始贏**(抵抗 ÷ 單異界產能 = 2,就是「一個門要幾異界」的刻度)。3 異界穩一門需 ~600s(10 分),4 異界 ~300s。
> **節奏待拍:** 這組偏「慢經營」(門 = 數分鐘)。要變「快循環」(門 = 數十秒),最乾淨的鈕是**砍基數**(基數 20 → 同樣 4 異界 60 秒穩一門,快 5 倍)。

**單位的一生(貢獻=戰力,同一個池):**
一個單位的「貢獻度」是一個會被消耗的池,看它先撞到誰:

- **路上無敵(正在贏):** 直達傳送門 → 灌入當下貢獻、推升進度 → **打 4 折**(生命/魔力扣一定比例)→ 從**泉水**復活、可再導向下一條線(變弱)。
  - **泉水(v0.3 定案,取代「從對門出來」):** 玩家放置的復活設施,一關一座。過門的單位統一從泉水復活(小幅回血、不回滿),沿泉水拉出的線再上工;沒放泉水(或折舊後低於榨乾線)則消散。回程因此是一條獨立的線 —— 未來的回復/加工設施都掛在線上。
- **路上遇敵(反噬中):** 與敵方數值對沖。對沖規則:**小的一方消滅、大的扣除後續行,相等同歸於盡**;被完全抵銷 → 死亡(對該門貢獻 0)。(數值大小僅示意規則;基準單位貢獻 = 1。)
- **折舊經濟:** 貢獻隨當前生命/魔力縮放,每趟過門打折(範例 4 折)→ 一隻約循環 2~3 趟即榨乾,靠**異界**源源生產補新血。這是本作「不靠錢」的資源線。
- **設施回血:** 單位路上經過**回復設施**會相對回血、貢獻相對回升,但**一次不回滿**,需多經幾座或繞圈補滿。路徑佈局多一層:安排單位「順路回血再上工」。(玩家蓋滿回血設施養滿單位 = 可行,平衡後再管。)

> **餵 = 推:** 一個單位的值,要嘛花在門上(餵)、要嘛花在敵人身上(推),同一池。差別只是當下淨值正負——正在餵、負在被推。整個遊戲收斂成一個動詞:**「把對的單位、夠快地,送到告急的那條線。」**
>
> **實作提醒:** 由於單位會「回血→再上工」,意味單位真的會**繞回重用**,那條較貴的「實體繞回」路由 MVP 多半就得做(不能只當折舊乘數);要不要進 MVP 晚點定。
>
> **支援設施類型** = 路徑上的「貢獻倍率」、「敵方減速區」、「回復設施」。**備選(日後):** 升級成輕量 lane auto-battler。MVP 不做。

### 角色特性

單位帶有若干「特性」標籤,用來和傳送門需求比對。初期分三類,**未來可擴充或成長出新分類**,故統稱「特性」而非固定三軸:

- **種族:** 妖精、龍、不死、野獸…
- **屬性:** 火、水、時…
- **體型:** 大 / 小。

### 經濟(MVP 暫緩)

- 先不做貨幣;資源視為無限(原型期可設 99999、花費皆 1)。
- **真正的約束 = 場地空間有限 + 敵方推力持續下壓征服進度 + 多門並行的規格差異。**
- 三者必須同時存在,否則玩法會漂成放置型。

### 分流

- MVP 階段:固定拉 2 條路徑、均分。自訂比例為後期再議。

## 四、MVP 範圍(獨立團隊的生死線)

### 要做(驗證核心)

- 單一場景。
- 2–3 個**起始異界**(自帶種族/屬性,當生產來源)。
- 3 個傳送門,規格需求為**單軸或雙軸**。
- 前線推拉 + 反噬。
- 穩定即過關。
- 裝置完整度:**1 次失敗即碎**。
- 最小的「獎勵點 → 永久解鎖」循環。

### 暫時不做

- 加工系統(屬局外解鎖,首版不做)。
- 貨幣經濟。
- 複雜分流(> 2 條 / 自訂比例)。
- 劇情與「真實身份」揭露。
- 大量種族/屬性內容。
- 異界那頭的視覺化(門內抽象、不演出)。

### 要回答的核心問題

> 「鋪產線餵門 + 反噬時即時重新分配」這個 **30 秒循環本身**好不好玩?
> 如果這個不抓人,加再多種族也救不回來。先做最小可玩版本驗證它。

## 六、技術設計(TDD,精簡)

> 多數欄位待填。先記下已知的架構決策,其餘等選型確定再補。

- **引擎 / 語言:** 待填(選型考量:2D 格狀場 + 路徑尋路 + 大量小單位,Unity 或 Godot 皆可勝任)。
- **場地模型:** 格狀棋盤;路徑沿格鋪設。
- **單位:** 沿路徑移動的實體,帶 `特性` 標籤 + `生命/魔力` 狀態 + `貢獻度池`;由**異界**產出、經過**加工**疊加標籤、經過**回復設施**部分回血。貢獻度 = f(當前生命/魔力);每趟過門打折(範例 4 折),回復設施可相對回升(不一次回滿)。
- **傳送門:** 單一數值軸,門檻為比例 **-3 崩潰 / -1 反噬 / 0 不穩定 / +3 穩定**,實際值 = 比例 × **基數(暫 100)**(單位貢獻暫 1)。達穩定 → 鎖定;到崩潰 → 關閉(失敗 +1)。「反噬」= 淨值為負、軸倒退的狀態,非獨立狀態機。
- **路徑對沖:** 我方單位與敵方單位在路上相遇即數值對沖(小者消滅、大者扣除續行,相等同滅)。存活單位抵達門 → 灌入剩餘貢獻;成功過門者從對門以折舊狀態返回、可重新導向(**單位繞回重用**)。
- **規格比對:** 單位標籤 vs 門需求,計算符合度(全符 > 部分符 > 不符)。**符合度同時就是該單位的貢獻/推力權重**(餵=推,共用一個數值)。
- **軸變化:** Δ進度 = Σ(抵達我方單位剩餘貢獻) − Σ(抵達敵方數值)。
- **存檔:** roguelike 永久解鎖與獎勵點。
- **第三方資源 / 授權:** 待填(上架時會用到,記得逐項記授權條款)。
- **建置 / 平台:** 待填。
